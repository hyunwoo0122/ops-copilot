## 🧠 반드시 외워야 할 것 3가지
================================================================

────────────────────────────────
1️⃣ Q = K = V = X (Self-Attention)
────────────────────────────────
쉽게 말하면:
  "나(X)를 가지고 나 자신(Q,K,V)을 분석한다"

왜 이렇게 해?
  외부 도움 없이 로그 시퀀스 스스로
  "나는 어떤 패턴이야?" 를 파악하기 위해서

코드로는:
  Q = K = V = X  ← 이 한 줄이 Self의 의미

────────────────────────────────
2️⃣ 재구성 오류 = 이상 점수
────────────────────────────────
쉽게 말하면:
  선생님이 정상 답안만 열심히 공부함
  → 정상 문제: 거의 정확하게 맞춤 → 오류 작음
  → 이상 문제: 본 적 없어서 틀림  → 오류 큼

코드로는:
  score = np.abs(X - output).mean()
  ← 입력과 출력의 차이가 클수록 이상

오늘 실제로 확인한 숫자:
  정상 → 0.2707 (작음 ✅)
  이상 → 0.4874 (큼 ✅)

────────────────────────────────
3️⃣ 슬라이딩 윈도우 O(n) 핵심
────────────────────────────────
쉽게 말하면:
  [1, 2, 3, 4, 5] 에서 크기 3짜리 창을 밀면서 볼 때
  매번 1+2+3, 2+3+4, 3+4+5 다시 더하면 느려
  → 앞 숫자 빼고, 뒤 숫자 더하면 한 번만 계산!

코드로는:
  window_sum += logs[num + k - 1]  # 뒤에서 들어온 값
  window_sum -= logs[num - 1]      # 앞에서 나간 값

속도 차이:
  매번 sum(): O(n×k) → 데이터 100개, k=10이면 1000번
  슬라이딩  : O(n)   → 데이터 100개면 무조건 100번

## 📌 헷갈리기 쉬운 것 정리
================================================================

❓ float('inf') 왜 써?
→ "아직 비교 안 했으니 일단 무한대로 시작"
→ 어떤 값이 와도 처음에는 무조건 갱신됨
→ result = 0 이나 result = 999 처럼 임의의 값 쓰면
   데이터에 따라 버그 날 수 있어

❓ threshold는 왜 함수 인자로 받아?
→ 실험마다 0.3, 0.5, 0.7 바꿔가며 테스트해야 해
→ 코드 안에 0.5 라고 박아두면 바꿀 때마다 코드 수정해야 함
→ Anti-Pattern: 하드코딩!

## 🇯🇵 면접 Q&A — 소리내서 읽어봐
================================================================

Q: Q=K=V=Xとするのはなぜですか？
A: 外部入力なしに、シーケンス自身が
   自分のトークン間の関係を計算するためです。
   (외부 입력 없이 시퀀스 스스로
    토큰 간 관계를 계산하기 위해서입니다.)

Q: 再構成誤差を異常スコアに使う理由は？
A: 正常データのみで学習したモデルは
   正常パターンを再構成できますが、
   異常データは再構成できません。
   本日の実験で正常0.27、異常0.49と確認しました。
   (정상 데이터만 학습한 모델은 정상은 재구성하지만
    이상은 재구성 못합니다. 오늘 실험에서 확인했습니다.)

Q: スライディングウィンドウがO(n)の理由は？
A: 毎回sum()を計算せず、先頭を引いて末尾を足すだけで
   各ステップO(1)で更新できるためです。
   (매번 합계를 구하지 않고 앞을 빼고 뒤를 더하는
    O(1) 업데이트만 하기 때문입니다.)
