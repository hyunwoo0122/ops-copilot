## 🧠 반드시 외워야 할 것
================================================================

────────────────────────────────
1️⃣ Early Stopping 구조 — 통째로 외워
────────────────────────────────
쉽게 말하면:
  "더 이상 좋아지지 않으면 일찍 멈추는 것"
  시험 공부를 8시간째에 점수가 안 오르면 자는 게 낫잖아

왜 필요해?
  너무 많이 학습 → 과적합(過学習)
  과적합 = 학습 데이터만 잘 맞추고
           새로운 데이터는 틀리는 현상

코드 구조 (순서 외워):
  best_loss = float('inf')   # 반드시 inf로 시작!
  patience_counter = 0

  for epoch in range(epochs):
      ... 학습 ...
      if loss < best_loss:
          best_loss = loss.item()
          patience_counter = 0
          torch.save(model.state_dict(), path)  # 최적 저장
      else:
          patience_counter += 1
          if patience_counter >= patience:
              break  # 종료!

────────────────────────────────
2️⃣ 모델 저장/불러오기
────────────────────────────────
쉽게 말하면:
  state_dict = 모델의 가중치만 뽑아낸 딕셔너리

저장:
  torch.save(model.state_dict(), "best_model.pt")

불러오기:
  model.load_state_dict(torch.load("best_model.pt"))

왜 model 통째로 저장하면 안 돼?
  torch.save(model) → 클래스 구조까지 저장
  → 나중에 코드가 바뀌면 불러오기 실패할 수 있어
  torch.save(model.state_dict()) → 가중치만 저장
  → 이식성(移植性)이 높아 → 실무 표준!

────────────────────────────────
3️⃣ best_loss 초기값이 중요한 이유
────────────────────────────────
  float('inf') → 어떤 loss도 처음엔 무조건 갱신
  0으로 하면  → 첫 loss가 0보다 크면 바로 Early Stop
  999로 하면  → loss가 999보다 크면 갱신 안 됨

  → 항상 float('inf') 로 시작!

## 카데인 알고리즘 (최대 연속 부분합)
================================================================

────────────────────────────────
핵심 아이디어
────────────────────────────────
쉽게 말하면:
  매 칸마다 선택지가 2가지야
  1) 지금까지 합에 현재 값을 더한다
  2) 현재 값부터 새로 시작한다
  → 둘 중 더 큰 것을 선택!

왜 -0.2를 버리고 0.8부터 시작해?
  [-0.2, 0.8, ...] → 합 시작 = 0.6
  [0.8, ...]       → 합 시작 = 0.8
  0.8 > 0.6 → -0.2 포함하면 손해! 버리는 게 나아

핵심 코드 (반드시 외워):
  current_sum = max(score, current_sum + score)
  max_sum = max(max_sum, current_sum)

손으로 따라가기:
  scores = [-0.2, 0.8, -0.3, 0.9, -0.5, 0.6]

  index 0: current=-0.2  max=-0.2
  index 1: current=max(0.8, -0.2+0.8)=0.8   max=0.8  ← 새로 시작
  index 2: current=max(-0.3, 0.8-0.3)=0.5   max=0.8
  index 3: current=max(0.9, 0.5+0.9)=1.4    max=1.4
  index 4: current=max(-0.5, 1.4-0.5)=0.9   max=1.4
  index 5: current=max(0.6, 0.9+0.6)=1.5    max=1.5
  정답: 1.5 ✅

시간복잡도: O(n)
  → 배열을 딱 한 번만 순회하기 때문에
  → 매 칸마다 max() 비교 O(1) × n번 = O(n)

## 🇯🇵 면접 Q&A — 소리내서 읽어봐
================================================================

Q: Early Stoppingを使う理由は？
A: 学習を続けすぎると過学習が起き、
   学習データには強いが新しいデータには
   正しく判断できなくなります。
   Early Stoppingは改善がなくなった時点で
   学習を止め、過学習を防ぎます。

Q: model.state_dict()を保存する理由は？
A: モデル全体を保存するより移植性が高く、
   コードが変わっても読み込み失敗のリスクが
   少ないためです。実務での標準的な方法です。

Q: カデインアルゴリズムがO(n)の理由は？
A: 配列を一度だけ走査し、
   各要素でmax()の比較をO(1)で行うため、
   全体でO(n)になります。

Q: numpy版とPyTorch版の再構成誤差の差は？
A: numpy版(学習なし)は1.8倍の差でしたが、
   PyTorchで学習後は42倍の差になりました。
   学習の効果で異常検知精度が大幅に向上しました。
