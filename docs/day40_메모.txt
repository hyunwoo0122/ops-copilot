6주차: Attention + LogBERT
────────────────────────────────
핵심:
  Q·K.T/√d_k → softmax → ·V
  Q=K=V=X (Self-Attention)
  재구성 오류 = 이상 점수

────────────────────────────────
7주차: PyTorch AutoEncoder
────────────────────────────────
핵심:
  nn.Module → super().__init__() 필수
  학습 루프 5단계 (zero_grad → backward → step)
  Early Stopping + torch.save(state_dict)
  model.eval() + torch.no_grad() 추론 시 필수
  정상 0.1646 / 이상 6.9236 (42배!)

────────────────────────────────
8주차: DataLoader + FPR/Recall
────────────────────────────────
핵심:
  TensorDataset + DataLoader(shuffle=True)
  batch[0] 으로 데이터 꺼내기
  epoch_loss = np.mean(batch_losses)
  dim=1 → 샘플별 오류
  FPR = FP / (FP + TN)
  성능 회귀 테스트 (F1 ≥ 0.8 기준선)
  최종: F1=0.995, FPR=0.000, Recall=0.990

## 🧠 반드시 외워야 할 패턴 모음
================================================================

────────────────────────────────
1️⃣ PyTorch 완전한 학습 루프
────────────────────────────────
dataset = TensorDataset(data)
loader  = DataLoader(dataset, batch_size=32, shuffle=True)

model.train()
for batch in loader:
    x = batch[0]
    optimizer.zero_grad()
    output = model(x)
    loss = criterion(output, x)
    loss.backward()
    optimizer.step()
    batch_losses.append(loss.item())

epoch_loss = np.mean(batch_losses)

────────────────────────────────
2️⃣ 추론 필수 패턴
────────────────────────────────
model.eval()
with torch.no_grad():
    scores = torch.mean(
        (data - model(data)) ** 2, dim=1
    ).numpy()

────────────────────────────────
3️⃣ Confusion Matrix 4가지
────────────────────────────────
TP: 이상 → 이상 (잘 잡음)
TN: 정상 → 정상 (잘 판단)
FP: 정상 → 이상 (오탐) → FPR = FP/(FP+TN)
FN: 이상 → 정상 (놓침) → Recall = TP/(TP+FN)

────────────────────────────────
4️⃣ 알고리즘 시간복잡도 총정리
────────────────────────────────
O(1)      : 딕셔너리 조회, 인덱스 접근
O(log n)  : 이진탐색
O(n)      : for문 한 번, DataLoader 순회
O(n log n): sorted(), 최소 구간 커버
O(n²)     : 이중 for문, LIS, threshold 탐지 통계

## 🇯🇵 면접 Q&A — 소리내서 읽어봐
================================================================

Q: このプロジェクトで最も重要な指標は？
A: 異常検知の運用観点では、FPRが最重要です。
   FPRが高いと正常ログへの誤検知が増え、
   担当者がアラートを無視するようになります。
   threshold=0.7でFPR=0.000を達成しました。

Q: 2ヶ月間で最も印象的な改善は？
A: numpy版(学習なし)では1.8倍の差でしたが、
   PyTorchで学習を追加して42倍に、
   DataLoaderとthresholdチューニングで
   最終的にF1=0.995・FPR=0.000を達成しました。

Q: 性能回帰テストとは何ですか？
A: コードが変更されても性能が一定水準以上を
   維持しているかを自動確認するテストです。
   F1≥0.8の基準線をpytestで実装しました。

Q: DataLoaderを使う理由は？
A: データをバッチに分割してメモリを節約し、
   shuffleで順序バイアスを防ぎます。
   大規模データでも学習可能になります。
