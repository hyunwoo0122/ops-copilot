## 🔥 실무에서 반드시 알아야 할 것
================================================================

────────────────────────────────
1️⃣ 학습 루프 5단계 — 평생 외워야 해
────────────────────────────────
실무에서 PyTorch 쓰는 모든 곳에 이 패턴이 있어:

model.train()                          # 학습 모드
optimizer.zero_grad()                  # 기울기 초기화
output = model(data)                   # 순전파
loss = criterion(output, data)         # loss 계산
loss.backward()                        # 역전파
optimizer.step()                       # 가중치 업데이트

❌ zero_grad() 빼먹으면 → 기울기 누적 → 학습 망가짐
❌ model.train() 빼먹으면 → dropout 꺼진 채 학습 → 결과 이상

────────────────────────────────
2️⃣ Early Stopping — 실무 필수
────────────────────────────────
쉽게 말하면:
  "더 이상 좋아지지 않으면 일찍 멈추기"

왜 필요해?
  너무 많이 학습 → 과적합(過学習)
  과적합 = 학습 데이터만 잘 맞추고
           새 데이터는 틀리는 현상

실무 패턴 (이 구조 외워):
  best_loss = float('inf')
  patience_counter = 0

  for epoch in range(epochs):
      ...학습...
      if loss < best_loss:
          best_loss = loss.item()
          patience_counter = 0
          torch.save(model.state_dict(), path)
      else:
          patience_counter += 1
          if patience_counter >= patience:
              break

────────────────────────────────
3️⃣ 모델 저장/불러오기 — 실무 표준
────────────────────────────────
저장:
  torch.save(model.state_dict(), "best_model.pt")

불러오기:
  model.load_state_dict(torch.load("best_model.pt"))

왜 model 통째로 저장하면 안돼?
  torch.save(model)              → ❌ 코드 바뀌면 불러오기 실패
  torch.save(model.state_dict()) → ✅ 가중치만 저장, 이식성 높음

────────────────────────────────
4️⃣ 추론할 때 반드시 써야 하는 것 2가지
────────────────────────────────
model.eval()               # Dropout 끔 → 결과 안정
with torch.no_grad():      # 기울기 계산 끔 → 메모리 절약
    output = model(data)

❌ eval() 없으면 → dropout이 켜진 채 추론 → 매번 결과 달라짐
❌ no_grad() 없으면 → 불필요한 메모리 낭비

────────────────────────────────
5️⃣ batch_size 개념 — 실무에서 항상 씀
────────────────────────────────
쉽게 말하면:
  데이터 전체를 한 번에 학습하면 메모리 부족
  → 조금씩 잘라서 학습하는 것 = batch

shape 표기:
  (100, 8) → 100개 데이터, 각 8차원
  (batch, 8) → 몇 개든 동작한다는 뜻 (더 정확한 표현!)

실무에서는:
  DataLoader 로 자동으로 batch 나눠줌
  → 나중에 8주차에 배울 예정

## 2개월차 전체 핵심 정리
================================================================

────────────────────────────────
5주차: MLflow + pytest 기반
────────────────────────────────
배운 것:
  mlflow.log_params()   → 파라미터 기록
  mlflow.log_metric()   → 결과 점수 기록
  pytest fixture        → 테스트 데이터 공유
  assert               → 조건 검증

────────────────────────────────
6주차: Attention + LogBERT
────────────────────────────────
배운 것:
  Q·K.T/√d_k → softmax → ·V  (Attention 공식)
  Q=K=V=X                     (Self-Attention)
  재구성 오류 = 이상 점수
  정상 0.2707 / 이상 0.4874   (1.8배)

────────────────────────────────
7주차: PyTorch AutoEncoder
────────────────────────────────
배운 것:
  nn.Module          → 모델 뼈대
  학습 루프 5단계    → 평생 쓰는 패턴
  Early Stopping     → 과적합 방지
  torch.save/load    → 모델 저장/불러오기
  tmp_path           → 테스트 파일 격리

핵심 결과:
  정상 0.1646 / 이상 6.9236   (42배!)
  numpy 대비 23배 향상

## 🇯🇵 면접 Q&A — 소리내서 읽어봐
================================================================

Q: このプロジェクトで最も印象的な改善は？
A: PyTorchで学習を追加することで、
   正常・異常の再構成誤差の差が
   1.8倍から42倍に改善されました。

Q: 2ヶ月間で最も成長した点は？
A: 1ヶ月目はIsolationForestのみでしたが、
   2ヶ月目はPyTorchでAutoEncoderをスクラッチ実装し、
   学習・Early Stopping・テストまで一貫して
   実装できるようになりました。

Q: 過学習をどう防ぎましたか？
A: Early Stoppingを実装し、
   lossが改善しない場合はpatience=3で
   学習を自動停止させました。
   また最良モデルをtorch.saveで保存しました。

Q: model.eval()とtorch.no_grad()の違いは？
A: eval()はDropoutを無効化して推論結果を安定させます。
   no_grad()は勾配計算を止めてメモリを節約します。
   推論時は必ず両方使います。
