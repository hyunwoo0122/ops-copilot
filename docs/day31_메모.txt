## 반드시 외워야 할 것
================================================================

────────────────────────────────
1️⃣ numpy vs PyTorch Tensor
────────────────────────────────
쉽게 말하면:
  numpy  = 계산기 (그냥 숫자 계산)
  PyTorch = 학습하는 계산기 (틀리면 스스로 고침)

결정적 차이 2가지:
  1) 자동 미분 (自動微分)
     → requires_grad=True 붙이면 역전파 자동으로 해줘
  2) GPU 연산
     → .to("cuda") 한 줄로 GPU로 이동 가능

코드로:
  numpy  : X = np.random.randn(5, 8)        # 학습 불가
  PyTorch: X = torch.randn(5, 8,
               requires_grad=True)           # 학습 가능!

────────────────────────────────
2️⃣ requires_grad=True 의미
────────────────────────────────
쉽게 말하면:
  "이 값은 학습으로 업데이트 해줘" 라고 PyTorch에 알리는 것

역전파(backprop) 때:
  requires_grad=True  → 기울기 자동 계산 → 가중치 업데이트
  requires_grad=False → 기울기 계산 안 함 → 업데이트 없음

실무에서:
  학습할 파라미터    → True  (모델 가중치)
  입력 데이터       → False (데이터는 학습 안 해도 됨)

────────────────────────────────
3️⃣ 역전파(backprop)
────────────────────────────────
쉽게 말하면:
  시험을 보고 틀린 문제를 확인해서
  "왜 틀렸는지" 를 역으로 추적해서 공부법을 고치는 것

공식:
  가중치 = 가중치 - 학습률 × 기울기
  (곱하는 게 아니라 빼는 거야!)

코드로:
  loss.backward()    # 역전파 실행 (기울기 자동 계산)
  optimizer.step()   # 가중치 업데이트

## 🔥 nn.Module — 이 부분이 제일 중요해
================================================================

────────────────────────────────
nn.Module이란?
────────────────────────────────
쉽게 말하면:
  모델을 담는 "그릇"이야.
  레이어(층)와 가중치를 한 곳에서 관리해주는 클래스.

직접 관리하면:
  weight1 = torch.randn(8, 8, requires_grad=True)
  weight2 = torch.randn(8, 4, requires_grad=True)
  weight3 = torch.randn(4, 1, requires_grad=True)
  # 가중치가 늘어날수록 추적이 엄청 복잡해짐 😱

nn.Module 쓰면:
  class MyModel(nn.Module):
      def __init__(self):
          super().__init__()
          self.layer1 = nn.Linear(8, 8)  # 가중치 자동 생성
          self.layer2 = nn.Linear(8, 4)
          self.layer3 = nn.Linear(4, 1)

  model = MyModel()
  model.parameters()  # 모든 가중치 한 번에 접근 ✅

────────────────────────────────
nn.Module이 자동으로 해주는 것 4가지
────────────────────────────────

1) 가중치 자동 관리
   → model.parameters() 로 전체 가중치 한 번에 접근
   → optimizer = torch.optim.Adam(model.parameters())
   → 가중치 하나씩 관리할 필요 없음

2) 학습/추론 모드 전환
   → model.train()  : 학습 모드 (dropout 등 활성화)
   → model.eval()   : 추론 모드 (dropout 비활성화)
   → 안 쓰면 테스트할 때 결과가 이상하게 나올 수 있어

3) 저장/불러오기
   → torch.save(model.state_dict(), "model.pt")  # 저장
   → model.load_state_dict(torch.load("model.pt"))  # 불러오기
   → 실무에서 모델 배포할 때 필수

4) 레이어 자동 등록
   → __init__에 self.xxx = nn.Linear() 로 선언하면
   → PyTorch가 자동으로 "이건 학습할 파라미터야" 로 인식

────────────────────────────────
nn.Module 기본 구조 — 외워둬
────────────────────────────────

class 모델이름(nn.Module):
    def __init__(self):
        super().__init__()       # ← 반드시 필요!
        self.레이어 = nn.Linear(입력, 출력)

    def forward(self, x):        # ← 순전파 정의
        x = self.레이어(x)
        return x

model = 모델이름()               # 모델 생성
output = model(input_data)      # forward() 자동 호출

────────────────────────────────
자주 쓰는 nn 레이어 3가지
────────────────────────────────

nn.Linear(in, out)
  → 선형 변환 (y = Wx + b)
  → 가장 기본, 제일 많이 씀

nn.MultiheadAttention(d_model, num_heads)
  → Transformer의 핵심
  → 6주차에 numpy로 만든 것의 PyTorch 버전!

nn.Embedding(vocab_size, d_model)
  → 단어/토큰을 숫자 벡터로 변환
  → LogBERT에서 로그 토큰을 벡터로 만들 때 사용

## 누적합 알고리즘 — 오늘 배운 것
================================================================

핵심 공식 (반드시 외워):
  prefix[0] = 0
  prefix[i] = prefix[i-1] + scores[i-1]
  구간[l, r] = prefix[r+1] - prefix[l]

왜 O(1)?
  미리 다 더해뒀으니까
  질의마다 뺄셈 1번만 하면 끝!

시간복잡도:
  prefix 만들기: O(n) → 1번만
  각 질의 답하기: O(1) × q개 = O(q)
  전체: O(n + q)  ← sum() 쓰면 O(n×q) 였던 것

## 🇯🇵 면접 Q&A — 소리내서 읽어봐
================================================================

Q: NumPyとPyTorchの違いは？
A: PyTorchのTensorは自動微分(requires_grad)と
   GPU演算をサポートしており、学習ループを
   簡単に実装できます。

Q: nn.Moduleを使う理由は？
A: モデルのパラメータを一元管理でき、
   学習・推論モードの切り替え、
   保存・読み込みが簡単にできるためです。

Q: 累積和がO(1)の理由は？
A: 事前に計算しておくことで、
   各クエリをprefix[r+1]-prefix[l]の
   引き算1回で答えられるためです。
