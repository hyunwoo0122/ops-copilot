## 🔥 반드시 알아야 할 것
================================================================

────────────────────────────────
1️⃣ dim=1 의미 — 헷갈리면 안 돼
────────────────────────────────
data shape = (1000, 8)
              ↑      ↑
            dim=0   dim=1

dim=0 → 열 방향 평균 → shape (8,)    ← 특징별 평균 (틀림)
dim=1 → 행 방향 평균 → shape (1000,) ← 샘플별 평균 (맞음)

샘플마다 이상 점수 1개가 필요 → dim=1 !

코드:
  scores = torch.mean((data - output) ** 2, dim=1).numpy()

────────────────────────────────
2️⃣ FPR 직접 계산 공식 — 외워
────────────────────────────────
FPR = FP / (FP + TN)
    = 정상인데 이상으로 잘못 탐지 / 전체 정상

코드:
  fp = np.sum((y_true == 0) & (y_pred == 1))  # 정상→이상 오탐
  tn = np.sum((y_true == 0) & (y_pred == 0))  # 정상→정상 정탐
  fpr = fp / (fp + tn) if (fp + tn) > 0 else 0.0

────────────────────────────────
3️⃣ 혼동 행렬(Confusion Matrix) — 4가지 외워
────────────────────────────────
              예측 정상  예측 이상
실제 정상  |   TN    |   FP   |  ← FPR = FP/(FP+TN)
실제 이상  |   FN    |   TP   |  ← Recall = TP/(TP+FN)

TP: 이상을 이상으로 정확히 탐지  ← 많을수록 좋아
TN: 정상을 정상으로 정확히 판단  ← 많을수록 좋아
FP: 정상인데 이상으로 오탐       ← 적을수록 좋아 (FPR)
FN: 이상인데 정상으로 놓침       ← 적을수록 좋아 (Recall)

────────────────────────────────
4️⃣ F1만 보면 안 되는 이유 — 면접 필수
────────────────────────────────
FPR이 높으면:
  정상 로그를 이상으로 잘못 탐지 → 오탐 알림 폭탄
  → 담당자가 알림 무시하게 됨
  → 진짜 이상을 놓칠 위험 높아짐

오늘 결과로 증명:
  threshold=0.05: F1=0.201, FPR=0.793 (알림 폭탄!)
  threshold=0.70: F1=0.995, FPR=0.000 (완벽!)

────────────────────────────────
5️⃣ threshold 선택 전략
────────────────────────────────
1) F1이 최대인 threshold 후보 선택
2) FPR도 확인 → 너무 높으면 다음 후보
3) F1과 FPR의 균형점을 최적값으로 결정

오늘 결과:
  threshold=0.7 → F1=0.995, FPR=0.000 → 최적! ✅

## 알고리즘 — 최적 threshold 정확도 탐색
================================================================

핵심 로직:
  threshold마다:
    정상 중 threshold 이하 개수 (정상 정탐)
    이상 중 threshold 초과 개수 (이상 정탐)
    accuracy = (정상 정탐 + 이상 정탐) / 전체

코드 구조:
  for threshold in thresholds:       # t번
      for i in range(len(normal)):   # n번
          if threshold > normal[i]: count += 1
      for j in range(len(anomaly)):  # m번
          if threshold < anomaly[j]: count += 1
      accuracy = count / (n + m)

시간복잡도:
  바깥 for문: t번
  안쪽 for문: n + m번
  전체: O(t × (n+m))
  t ≈ n+m 이면 → O(n²)

## 🇯🇵 면접 Q&A
================================================================

Q: F1スコアだけでなくFPRも確認する理由は？
A: FPRが高いと正常ログへの誤検知が増え、
   担当者がアラートを無視するようになります。
   本日の実験でthreshold=0.05のときFPR=0.793でしたが、
   threshold=0.70ではFPR=0.000を達成しました。

Q: dim=1でサンプルごとの誤差を計算する理由は？
A: データのshapeが(n_samples, 8)のとき、
   dim=1で平均するとshape(n_samples,)になり、
   サンプルごとの異常スコアが得られます。
   dim=0だと特徴量ごとの平均になってしまいます。

Q: 最適なthresholdをどう選びましたか？
A: threshold=0.1〜1.0で7パターン実験し、
   F1スコアが最大のthreshold=0.7を選択しました。
   このときF1=0.995、FPR=0.000を達成しています。
