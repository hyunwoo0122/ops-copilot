## 오늘 배운 핵심 개념

[Attention]
- 각 토큰(로그)이 다른 모든 토큰과의 관련성 점수를 계산해서
  중요한 부분에 더 집중하는 방법
- IsolationForest: 각 포인트를 독립적으로 평가
- Transformer  : 시퀀스 전체 문맥(context)을 함께 봄
  → 로그 흐름 속 이상 패턴을 더 정확하게 탐지 가능

[Self-Supervised vs 지도학습]
- 지도학습      : 정답(레이블) 있음 → 이상=1, 정상=0
- Self-Supervised: 정답 없음, 스스로 만들어서 학습
                   예: 로그를 가리고 가린 부분 맞추기

[LogBERT 이상탐지 방법]
- 정상 로그만으로 학습
- 이상 로그가 오면 예측 오류가 커짐
- 예측 오류 크기를 이상 점수로 사용

## FPR vs Recall 직접 계산
- 전체: 정상 90개, 이상 10개
- TP=8, FN=2, FP=5, TN=85

Recall = TP / (TP + FN) = 8/10 = 0.8  (80%)
FPR    = FP / (FP + TN) = 5/90 = 0.055 (5.5%)

실무 해석:
- 하루 1000개 로그 중 정상 900개 → 5.5% = 49개 오탐
- 담당자가 49개 가짜 알림을 매일 확인 → 알림 무시하게 됨
- F1만 보면 안 되는 이유 = FPR이 높으면 실무에서 못 씀

## Anti-Pattern — 오늘 배운 것
- F1만 보고 모델 선택 → FPR도 반드시 확인
- 임계값 고정 → 데이터 분포 바뀌면 자동 조정 필요

## 오늘의 알고리즘 — 슬라이딩 윈도우

[문제]
크기 k인 연속 구간 중 이상(1) 비율이 0.5 초과인 구간의 시작 인덱스 반환

[두 버전 비교]
O(n*k) 버전: sum()을 매번 새로 계산
O(n)   버전: 앞을 빼고 뒤를 더하는 방식 (슬라이딩 윈도우)

# O(n*k) — 직관적이지만 k가 클수록 느려짐
window_sum = sum(logs[index:index+k])  # 매번 k번 계산

# O(n) — 슬라이딩 윈도우
window_sum += logs[index + k - 1]  # 뒤에서 들어온 값
window_sum -= logs[index - 1]       # 앞에서 빠진 값

핵심: k값이 달라도 "앞 빼고 뒤 더하기" 규칙은 항상 성립

[시간복잡도]
- O(n*k): for문 O(n) × sum() O(k) = O(n*k)
- O(n)  : 첫 sum() O(k) + 이후 더하기/빼기 O(1) × n = O(n)

## 면접 대비 Q&A

Q: IsolationForestとTransformerベースの違いは？
A: IsolationForestは各データポイントを独立して評価しますが、
   Transformerはシーケンス全体の文脈を考慮します。
   そのため、ログの流れの中での異常パターンを
   より正確に検知できます。

Q: FPRとRecallのトレードオフは？
A: 閾値を上げるとFPRは下がりますが、Recallも下がります。
   運用環境ではFPRが高いとアラートが増え、
   担当者がアラートを無視するようになります。

Q: スライディングウィンドウのO(n)解法は？
A: 毎回sum()を計算する代わりに、
   前の値を引いて後ろの値を足すことで
   O(1)の更新が可能になり、全体O(n)になります。
