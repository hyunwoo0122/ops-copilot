## 🧠 반드시 외워야 할 것 3가지
================================================================

────────────────────────────────
1️⃣ threshold 결정 방법
────────────────────────────────
쉽게 말하면:
  경보기 감도를 어떻게 설정하느냐의 문제야

결정 순서:
  1) threshold를 여러 값으로 바꿔가며 실험
  2) F1이 가장 높은 값을 후보로 선택
  3) FPR도 확인 → 너무 높으면 오탐 폭탄
  4) F1과 FPR의 균형점을 최적값으로 결정

오늘 결과:
  threshold=0.1 → F1=0.33 (낮은 이유: 학습 없는 프로토타입)
  진짜 LogBERT는 학습으로 정상/이상 점수 차이를 키워서 F1이 높아짐

────────────────────────────────
2️⃣ fixture 쓰는 기준
────────────────────────────────
쉽게 말하면:
  "여러 테스트가 같은 데이터를 쓰면 fixture"
  "혼자만 쓰는 데이터면 함수 안에서 직접 생성"

오늘 예시:
  test_anomaly_score_higher  → fixture 사용 (데이터 공유)
  test_score_non_negative    → fixture 사용 (데이터 공유)
  test_threshold_effect      → fixture 없음
                               (threshold 변화만 테스트,
                                데이터는 함수 안에서 직접 생성)

────────────────────────────────
3️⃣ 그리디 알고리즘 — 최소 알림 횟수
────────────────────────────────
쉽게 말하면:
  "지금 커버 못한 이벤트를 만나면 바로 알림 보내기"
  → 항상 지금 당장 최선의 선택을 해나가는 방식 = 그리디

핵심 코드 (반드시 외워!):
  cover_until = -1
  for event in sorted(events):
      if event > cover_until:       # 커버 안 됨
          count += 1                # 알림 발송
          cover_until = event + k - 1  # 커버 범위 갱신

시간복잡도:
  sorted() → O(n log n)
  for문    → O(n)
  전체     → O(n log n)  ← 큰 쪽이 대표

## 📌 헷갈리기 쉬운 것 정리
================================================================

❓ event + k vs event + k - 1
  k=4, event=1 이면:
  event + k     = 5 → 1,2,3,4,5 를 커버 (5분까지)
  event + k - 1 = 4 → 1,2,3,4 를 커버 (4분까지)
  문제 정의에 따라 달라지므로 예시로 직접 확인할 것!

❓ F1=0.33이 낮은 이유
  정상(0.27)과 이상(0.49) 점수 차이가 작아서
  → 학습 없는 프로토타입이라 정상
  → 진짜 LogBERT는 학습으로 이 차이를 키움

❓ O(n log n) vs O(n) 구분
  O(n)      : for문 한 번 순회
  O(n log n): sorted() 정렬
  O(log n)  : 이진탐색 (Day 27)
  큰 쪽이 전체 복잡도를 대표함

## 🇯🇵 면접 Q&A — 소리내서 읽어봐
================================================================

Q: 最適なthresholdをどうやって決めますか？
A: F1スコアが最大になるthresholdを探した上で、
   FPRも確認します。FPRが高いと誤検知が増え、
   担当者がアラートを無視するようになるためです。

Q: fixtureを使うテストと使わないテストの違いは？
A: 複数のテストで同じデータを共有する場合はfixture、
   そのテスト専用のデータは関数内で直接生成します。

Q: グリーディアルゴリズムとは何ですか？
A: 各ステップで最善の選択をし続けるアルゴリズムです。
   今回は「カバーできないイベントに出会ったら
   すぐにアラートを送る」という選択を繰り返しました。

Q: このアルゴリズムの時間計算量は？
A: sorted()がO(n log n)、forループがO(n)なので、
   全体はO(n log n)です。
