1️⃣ DataLoader 실무 패턴 — 통째로 외워
────────────────────────────────
from torch.utils.data import DataLoader, TensorDataset

dataset = TensorDataset(data)              # 데이터 포장
loader  = DataLoader(                      # 배치 분배기
    dataset,
    batch_size=32,    # 한 번에 32개씩
    shuffle=True,     # 매 epoch 순서 섞기
)

for epoch in range(epochs):
    model.train()
    batch_losses = []
    for batch in loader:         # 32개씩 자동으로!
        x = batch[0]             # TensorDataset은 튜플로 반환
        optimizer.zero_grad()
        output = model(x)
        loss = criterion(output, x)
        loss.backward()
        optimizer.step()
        batch_losses.append(loss.item())
    epoch_loss = np.mean(batch_losses)  # epoch 평균 loss

────────────────────────────────
2️⃣ batch[0] 으로 꺼내는 이유
────────────────────────────────
TensorDataset은 튜플로 반환해:
  batch = (x_tensor,)   ← 튜플!

그래서:
  x = batch[0]   ✅  첫 번째 원소
  x = batch      ❌  튜플 전체가 들어가서 에러

────────────────────────────────
3️⃣ epoch loss를 평균내는 이유
────────────────────────────────
마지막 배치는 batch_size보다 작을 수 있어:
  1000개 ÷ 32 = 31배치(992개) + 나머지 8개

마지막 loss만 쓰면:
  8개짜리 작은 배치의 값만 기록 → 대표성 없음

평균을 쓰면:
  전체 배치의 학습 상태를 정확하게 표현 ✅

────────────────────────────────
4️⃣ 7주차 vs 8주차 비교 — 면접 포인트
────────────────────────────────
7주차:
  model(normal_data)    # 100개 전체 한 번에
  → 데이터 늘리면 메모리 부족

8주차:
  for batch in loader:  # 32개씩 자동 분배
      model(batch[0])
  → 1000개, 10만개도 문제없음!

## 알고리즘 — 배치별 threshold 초과 찾기
================================================================

핵심 패턴:
  range(0, len(scores), k)  → k씩 건너뜀
  sum(scores[i:i+k])        → 배치 합
  i // k                    → 배치 번호

코드:
  for i in range(0, len(scores), k):
      if sum(scores[i:i+k]) > threshold:
          return i // k
  return -1

시간복잡도:
  for문: O(n/k) 번
  sum() : O(k)
  전체  : O(n/k × k) = O(n)

DataLoader와의 연결:
  DataLoader도 내부적으로 이 패턴으로 배치를 나눠줘!

## 🇯🇵 면접 Q&A
================================================================

Q: DataLoaderを使う理由は？
A: データを自動的にバッチに分割し、
   メモリを節約しながら学習できます。
   shuffleで順序バイアスも防げます。

Q: バッチ学習でlossを平均する理由は？
A: 最後のバッチはbatch_sizeより小さい場合があり、
   そのlossだけではepoch全体を代表できません。
   全バッチの平均でepoch全体の学習状態を表します。

Q: なぜbatch[0]でデータを取得するのですか？
A: TensorDatasetはデータをタプルで返すため、
   最初の要素をbatch[0]で取得します。
