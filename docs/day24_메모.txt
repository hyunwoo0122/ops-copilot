## 핵심 개념 — 격리성(隔離性)
- 전역변수 공유 → 테스트끼리 데이터 오염 가능 → 독립성 손실
- 인자로 전달  → 각 테스트가 자기만의 데이터 → 격리 보장
- fixture      → 중복 없이 + 독립성 동시 해결

## Anti-Pattern — 오늘 피한 것
- assert 없는 테스트  : 실행만 하고 확인 안 하면 테스트가 아님
- 테스트 안에서 print : assert로 증명해야 함
- 전역 데이터 의존    : 테스트마다 독립된 데이터 필요

## 면접 대비 Q&A
Q: なぜfixtureを使うのですか？
A: データ生成コードの重複を排除しながら、
   各テストに独立したデータを提供するため。
   グローバル変数では独立性が失われる。

Q: なぜ引数としてデータを渡すのですか？
A: グローバル変数を使うとテスト間でデータが共有され、
   独立性(隔離性)が失われるため。

Q: テストの種類を教えてください。
A: 正常ケース・境界値・エッジケースの3種類。

## 오늘의 알고리즘 — 괄호 유효성 검사
- 자료구조 : 스택 (list를 스택으로 사용)
- 핵심 로직:
  "(" 만나면 stack.append()
  ")" 만나면 stack이 비어있으면 False, 아니면 stack.pop()
  루프 끝나고 len(stack) == 0 이면 True

- 배운 것 1: "(" in stack → if stack 으로 바꾸면 O(n) → O(1)
  스택에 "("만 들어가도록 설계하면 내용 확인이 필요없음

- 배운 것 2: 인자명과 함수 안에서 쓰는 변수명이 달라서 생긴 버그
  def f(s): 로 받았는데 안에서 test를 쓰면 전역변수 참조

- 시간복잡도: O(n) → 문자열 한 번 순회
- 공간복잡도: O(n) → 최악의 경우("(((") 스택에 n개 쌓임

Q: 時間・空間計算量は？
A: 文字列を一度走査するのでO(n)、スタックに最大n個積まれるので空間もO(n)
